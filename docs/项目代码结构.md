# 项目代码结构

以下是当前精简版（`xhs + x`）的核心目录说明：

```text
EnergyCrawler
├── api/                      # WebUI API 与静态资源入口
├── base/                     # 抽象基类
├── cache/                    # 本地/Redis 缓存实现
├── cmd_arg/                  # CLI 参数解析
├── config/                   # 全局配置与数据库配置
├── database/                 # ORM 模型与会话管理
├── docs/                     # 文档
├── energy-service/           # Energy gRPC 服务（Go）
├── energy_client/            # Energy Python 客户端
├── libs/
│   └── stealth.min.js        # 浏览器反检测脚本
├── media_platform/
│   ├── xhs/                  # 小红书平台实现
│   └── twitter/              # X(Twitter) 平台实现
├── model/
│   └── m_xiaohongshu.py      # xhs URL/数据结构模型
├── proxy/                    # 代理池与供应商适配
├── scripts/                  # 运维/登录态/联调脚本
├── store/
│   ├── xhs/                  # xhs 存储实现
│   ├── twitter/              # x 存储实现
│   └── excel_store_base.py   # Excel 通用存储
├── tests/                    # 自动化测试
├── tools/                    # 浏览器/文件/通用工具
├── main.py                   # CLI 主入口
└── var.py                    # 运行期上下文变量
```

## 说明

- 当前仅保留 `xhs` 和 `x` 两个平台实现。
- 旧平台相关目录已移除，仅保留 `xhs` 与 `x`。
- 若要扩展新平台，建议参照 `media_platform/twitter` + `store/twitter` 的组织方式新增。
