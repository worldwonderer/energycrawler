# 常见问题

## 1. xhs 签名失败 / 接口返回风控

- 确认 Energy 服务已启动并监听 `localhost:50051`
- 确认 `config/base_config.py` 中：
  - `ENABLE_ENERGY_BROWSER = True`
  - `XHS_ENABLE_ENERGY = True`
- 确认账号在浏览器里可正常访问小红书页面

## 2. x 抓取鉴权失败

- 检查 `TWITTER_AUTH_TOKEN` 是否有效
- 必要时补充 `TWITTER_CT0`
- 建议先在浏览器里验证账号状态

## 3. `--platform` 参数报错

当前只支持：

- `xhs`
- `x`

## 4. 登录后仍不断验证

- 将 `HEADLESS` 设为 `False`，观察浏览器实际页面
- 清理对应平台登录态目录后重试

## 5. 数据文件为空

优先检查：

- 关键词或 ID 是否有效
- 登录是否成功
- 是否触发平台频控

## 6. Excel 文件没有生成

- 确认 `SAVE_DATA_OPTION = "excel"`
- 确认本次任务确实抓到了数据
- 任务结束后检查 `data/{platform}/` 输出目录

## 7. setup / config show / doctor 怎么用？

推荐顺序（可直接复制）：

```bash
# 1) setup（一键向导）
uv run energycrawler init --template .env.quickstart.example --check

# 2) config show（当前版本通过 API 查看）
curl -s http://localhost:8080/api/config/platforms | jq .
curl -s http://localhost:8080/api/config/options | jq .

# 3) doctor（一键体检）
uv run energycrawler doctor
```

若 `doctor` 失败，可按以下顺序排查：

1. 执行 `uv run energycrawler energy ensure`，确认 Energy 服务可用；
2. 执行 `uv run energycrawler auth status --json`，确认登录态；
3. 必要时执行 `uv run energycrawler doctor --skip-login-check --json`，先隔离服务连通问题。
