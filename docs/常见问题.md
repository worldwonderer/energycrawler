# 常见问题

## 1. xhs 签名失败 / 接口返回风控

- 确认 Energy 服务已启动并监听 `localhost:50051`
- 确认 `config/base_config.py` 中：
  - `ENABLE_ENERGY_BROWSER = True`
  - `XHS_ENABLE_ENERGY = True`
- 确认账号在浏览器里可正常访问小红书页面

## 2. x 抓取鉴权失败

- 检查 `TWITTER_AUTH_TOKEN` 是否有效
- 必要时补充 `TWITTER_CT0`
- 建议先在浏览器里验证账号状态

## 3. `--platform` 参数报错

当前只支持：

- `xhs`
- `x`

## 4. 登录后仍不断验证

- 将 `HEADLESS` 设为 `False`，观察浏览器实际页面
- 清理对应平台登录态目录后重试

## 5. 数据文件为空

优先检查：

- 关键词或 ID 是否有效
- 登录是否成功
- 是否触发平台频控

## 6. Excel 文件没有生成

- 确认 `SAVE_DATA_OPTION = "excel"`
- 确认本次任务确实抓到了数据
- 任务结束后检查 `data/{platform}/` 输出目录

## 7. quickstart / setup / doctor 怎么用？

推荐顺序（可直接复制）：

```bash
# 1) quickstart（首选）
uv run energycrawler quickstart

# 2) 运行态快照
uv run energycrawler status

# 3) doctor（可单独复检）
uv run energycrawler doctor
```

若当前分支还没有 `quickstart` 子命令，使用兼容路径：

```bash
uv run energycrawler init --template .env.quickstart.example --check
uv run energycrawler energy ensure
uv run energycrawler auth xhs-open-login --api-base http://localhost:8080
uv run energycrawler config env --mode core
uv run energycrawler doctor
```

若 `doctor` 失败，可按以下顺序排查：

1. 执行 `uv run energycrawler energy ensure`，确认 Energy 服务可用；
2. 执行 `uv run energycrawler auth status --json`，确认登录态；
3. 若 `xhs` 登录仍异常，执行 `uv run energycrawler auth xhs-open-login --api-base http://localhost:8080`（open + sync + verify）；
4. 执行 `uv run energycrawler status --json` 复核运行态；
5. 必要时执行 `uv run energycrawler doctor --skip-login-check --json`，先隔离服务连通问题。

如果你只想最短路径运行，直接：

```bash
uv run energycrawler quickstart
uv run energycrawler status
uv run energycrawler run --platform xhs --keywords 新能源
uv run energycrawler data list --platform xhs --limit 20
uv run energycrawler data latest --download
```

## 8. 缺少可选依赖怎么办？（Excel / Postgres）

按能力安装可选依赖：

```bash
# Excel 导出支持
uv sync --extra excel

# PostgreSQL 存储支持
uv sync --extra postgres

# 一次安装全量（兼容旧方式）
uv pip install -r requirements.txt
```

如你只需要最小抓取路径，可继续使用 core 依赖：

```bash
uv sync
# 或
uv pip install -r requirements-core.txt
```

## 9. `data list / data latest` 提示 API 不可达或没有可用文件

先确认 API 与数据状态：

```bash
# 1) CLI 运行态快照
uv run energycrawler status --json

# 2) API 是否可达（运行态健康）
curl -s http://localhost:8080/api/health/runtime | jq .

# 3) 是否确实有导出文件（CLI + API）
uv run energycrawler data list --platform xhs --limit 20
curl -s http://localhost:8080/api/data/files | jq '.data.files[:5]'
```

常见原因：

- API 端口不是 `8080`（请替换命令中的地址）；
- 任务尚未跑完，`data/` 目录暂时没有可用导出文件；
- 过滤条件过严（如 `platform=x` 但当前只有 `xhs` 导出）。
